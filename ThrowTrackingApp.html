Throw Tracking App

MongoDB and Data Charting,
RESTful API Design, Front End Design

Solo Project

Javascript, MERN Stack,
RESTful API, Chart.js

If you haven't read my about me section, I am a National level trakc and field athlete and I noticed that even the highest level of throwers still use very inefficient data tracking tools (like pen and paper), and I want to change that. Before starting this project, I had little to no experience with any of the MERN stack components or JS. I threw myself in and am learning as I go. This is a MERN stack web app for storing, tracking, and charting throwing data from track and field. It is a full stack app that can create, update, delete, and read data from the MongoDB database using a RESTful API, and displays data helpful for training via an admin dashboard such as different throw distances at meets, minimum and maximum throw distance from recorded meets and practices as well as the amount of meets and practices. Chart data can be filtered through using the type of throw, which type of throw i.e. first throw of each meet, or at practice standing throws, and the date of the meet or practice. I am learning a lot from developing this app and plan to continue developing it, and using it for tracking my throwing data.
